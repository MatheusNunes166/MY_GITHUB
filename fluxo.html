1

Clonar o Repositório

Para começar a trabalhar em um projeto existente, primeiro você precisa clonar o repositório para sua máquina local.

git clone https://github.com/usuario/repositorio.git
# Isso cria uma cópia local do repositório
2

Criar uma Branch

Antes de fazer alterações, crie uma nova branch para isolar seu trabalho. Isso evita conflitos com a branch principal.

git checkout -b minha-nova-feature
# Cria e alterna para uma nova branch
3

Fazer Alterações e Commits

Faça suas alterações no código e salve-as com commits. Cada commit deve representar uma mudança lógica e autocontida.

git add .
# Adiciona todas as alterações para o próximo commit
git commit -m "Descrição significativa das alterações"
# Registra as alterações com uma mensagem descritiva
4

Sincronizar com o Repositório Remoto

Antes de enviar suas alterações, é uma boa prática puxar as alterações mais recentes do repositório remoto para evitar conflitos.

git pull origin main
# Atualiza sua branch local com as alterações mais recentes
5

Enviar Alterações para o Repositório Remoto

Após confirmar suas alterações, envie-as para o repositório remoto no GitHub.

git push origin minha-nova-feature
# Envia sua branch para o repositório remoto
6

Abrir um Pull Request

No GitHub, abra um Pull Request para solicitar que suas alterações sejam mescladas na branch principal.

Isso permite revisão de código, discussão e testes antes da integração.

7

Revisão e Mesclagem

Após a revisão e aprovação do código, o Pull Request pode be mesclado. Isso integra suas alterações à branch principal.

git checkout main
# Alterna para a branch principal
git merge minha-nova-feature
# Mescla as alterações da sua branch
